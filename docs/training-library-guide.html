Adding Content to the Training Library

A quick guide for coaches & staff — how to get videos, drills, and guides into the dashboard.

📎

First, the key idea: the Training Library is link-based, not a file uploader. You host the content somewhere (YouTube, Vimeo, Google Drive, Google Docs) and paste the link. The dashboard then plays it inline or opens it in a new tab. There is no "upload a file" button — and that's on purpose: it keeps the library fast and nothing gets lost.

1.Add a resource — step by step

  1. 1
    Open the dashboard & sign in.

    Go to lbsocceracademy.org/dashboard/ and sign in with an approved Google account.

  2. 2
    Click “Training” in the left sidebar.

    This opens the Training Library view with all current videos, guides, and session plans.

  3. 3
    Click “Add resource” (top-right).

    A form slides in from the side.

  4. 4
    Fill in the form.

    Title and Link are required. Type = Video, Guide, Session plan, or Link. Then optionally add Length (“6 min · 1 page”), tick the Age groups (K–2 · U6–U8, 3–5 · U9–U11, 6–8 · U12–U15, or All ages), tick Topics, and write a short Description.

  5. 5
    Click “Add resource.”

    The card appears in the library right away. To change it later, click the card and choose Edit or Delete.

2.Where to put the actual file

The Link field needs a URL. Host the file first, then paste its share link:

Only YouTube and Vimeo links embed and play in-dashboard. Everything else opens externally — which is perfectly fine for PDFs and docs.

3.Good to know

Start from the examples

The library ships with 8 starter entries (the Eagles Way guides + sample sessions). If it's empty, click “Load starter resources” to drop them in — then copy their format for your own.

Sharing across all coaches (the sync)

Resources save to your browser instantly and are built to sync to a Resources tab in the program's Google Sheet, so every signed-in coach sees the same library. That sync only works once the Apps Script backend has been re-deployed and its setup() run. Until then, entries live only on the device that added them.
